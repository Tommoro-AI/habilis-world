# 2.1 신규 디바이스 등록

1. 좌측 메뉴 바에서 **Add Robot** 탭을 클릭합니다.
    
    [이미지]
    
2. Config 페이지의 Network 탭에서 아래 항목을 작성합니다.
    
    <aside>
    🛠
    
    > **[ 선택 ]**
    > `AI SERVER IP` 입력 필드는 ai 추론 및 학습 기능을 사용할 경우에만 작성합니다.
    > 
    > - `NETWORK NAME` : 네트워크 이름 (원하는 이름으로 작성)
    > - `ROBOT IP` : 신규 디바이스의 IP
    > - `AI SERVER IP` : ai-manager 확장팩이 설치된 PC의 IP 주소
    > 
    > [이미지]
    > 
    </aside>
    
    <aside>
    📖
    
    > **[ 참고 ] 디바이스 IP를 모르는 경우 (Linux 버전)**
    > 로봇에 모니터와 키보드가 연결되어 있다면 직접 확인이 가능합니다.
    > 1. **터미널 실행 : 터미널(Terminal)** 창을 엽니다. (`Ctrl`+`Alt`+`T`)
    > 2. **명령어 입력 :** `ifconfig` 명령어를 입력합니다.
    > 3. **IP 확인 :** 사용 중인 네트워크 인터페이스(예: `eth0`, `wlan0`)의 **inet** 옆 숫자를 확인합니다.
    > **주의:** `lo` (Local Loopback) 항목의 `127.0.0.1`은 로봇의 실제 IP가 아닙니다.
    > 
    </aside>
    
    <aside>
    📖
    
    > **[ 참고 ] 디바이스 IP를 모르는 경우 (MacOS 버전)**
    > 로봇(또는 해당 디바이스)에 모니터와 키보드가 연결되어 있다면 직접 확인이 가능합니다.
    > 1. **터미널 실행:** `Command(⌘)` + `Space`를 눌러 **Terminal**을 검색해 실행합니다.
    > 2. **명령어 입력:** `ifconfig` 또는 `ipconfig getifaddr en0` 명령어를 입력합니다.
    > 3. **IP 확인**
    > 
    > - `ifconfig` 입력 시: 사용 중인 네트워크 인터페이스(예: en0, en1) 항목의 **inet** 옆 숫자를 확인합니다.
    > - `ipconfig getifaddr en0` 입력 시: 즉시 할당된 IP 주소가 출력됩니다.
    > 
    > **주의:** `lo0` 항목의 `127.0.0.1`은 기기 자체의 루프백 주소이므로 실제 네트워크 IP가 아닙니다.
    > 
    </aside>
    
    <aside>
    📖
    
    > **[ 참고 ] 디바이스 IP를 모르는 경우 (Windows 버전)**
    > 로봇(또는 해당 디바이스)에 모니터와 키보드가 연결되어 있다면 직접 확인이 가능합니다.
    > 1. **명령 프롬프트 실행:** `Win` + `R` 키를 누른 후 **cmd**를 입력하고 엔터를 누릅니다.
    > 2. **명령어 입력:** `ipconfig` 명령어를 입력합니다.
    > 3. **IP 확인 :** 사용 중인 어댑터(예: 이더넷 어댑터 또는 무선 LAN) 항목에서 **IPv4 주소** 옆의 숫자를 확인합니다.
    > 
    > 
    > **주의:** `기본 게이트웨이` 주소와 혼동하지 않도록 주의하세요. 또한, 리눅스와 마찬가지로 `127.0.0.1`은 실제 외부 접속용 IP가 아닙니다.
    > 
    </aside>

<aside>
💡

> **[중요] 🟡 노란색 표시: 연결 중 (Connection Pending)**
> 
> - **상태:** `robot_manager`에 요청을 보냈으나 아직 응답이 오지 않은 상태입니다.
> - **대응:** 통신 지연일 수 있으므로 **최소 30초 정도** 기다려 주세요.
> - **해결:** 30초가 지난 후에도 계속 노란색이라면, 로봇(기기)의 전원을 껐다가 다시 켜주세요.
>     
>     [이미지]
</aside>

<aside>
💡

> **[중요] 🔴 빨간색 표시: 연결 실패 (Disconnected)**
> 
> - **상태:** `robot_manager` 요청조차 보내지 못하는 상태로, 로봇이 꺼져 있거나 네트워크 연결이 완전히 끊긴 것으로 판단됩니다.
> - **해결:** 로봇 기기의 전원 및 네트워크 상태를 확인하고, 로봇을 아예 다시 껐다가 켜주세요.
>     
>     [이미지]
</aside>
