:::info[정보]
### 디바이스 IP를 모르는 경우

로봇(또는 해당 디바이스)에 모니터와 키보드가 연결되어 있다면 직접 확인이 가능합니다.

### 리눅스
1. **터미널(Terminal)** 창을 엽니다. (`Ctrl` + `Alt` + `T`)
2. `ifconfig` 명령어를 입력합니다.
3. 사용 중인 네트워크 인터페이스(예: `eth0`, `wlan0`)의 **inet** 옆 숫자를 확인합니다.

### 맥OS
1. `Command(⌘)` + `Space`를 눌러 **Terminal**을 검색해 실행합니다.
2. `ifconfig` 또는 `ipconfig getifaddr en0` 명령어를 입력합니다.
3. `ifconfig` 입력 시, 사용 중인 네트워크 인터페이스(예: en0, en1) 항목의 **inet** 옆 숫자를 확인합니다.
   또는 `ipconfig getifaddr en0` 입력 시, 즉시 할당된 IP 주소가 출력됩니다.

### 윈도우
1. `Win` + `R` 키를 누른 후 **cmd**를 입력하고 엔터를 누릅니다.
2. `ipconfig` 명령어를 입력합니다.
3. 사용 중인 어댑터(예: 이더넷 어댑터 또는 무선 LAN) 항목에서 **IPv4 주소** 옆의 숫자를 확인합니다.
:::

:::warning[주의]
`127.0.0.1`은 외부 접속용 IP가 아닙니다.
:::

## Step 3. 인증서 허용    
통신 보안을 위해 최초 1회 인증서 승인이 필요합니다.

1. 우측 상단의 **[Trust ROS Cert]** 버튼을 클릭합니다. 
![console_config_trustROS](./image/2.1_console_config_trustROS_click0.png)
    
2. 브라우저 새 창에 보안 경고가 뜨면 '**고급 > 안전하지 않음으로 이동(Proceed)**'을 선택합니다.
![console_config_trustROS](./image/2.1_console_config_trustROS_click1.png)
![console_config_trustROS](./image/2.1_console_config_trustROS_click2.png)    

3. 인증서 페이지가 정상적으로 열리면 창을 닫고 콘솔로 돌아옵니다.     
![console_config_trustROS](./image/2.1_console_config_trustROS_click3.png)

## Step 5. 로봇 전원 구동

1. <strong>ROBOT MODEL</strong> 필드에 로봇 모델 이름(예: `rby1a`)이 정상적으로 표시되는지 확인합니다.
2. 우측의 **[ROBOT ON]** 버튼을 클릭하여 시스템 전원을 켭니다.
    ![console_config_trustROS_retry](./image/2.1_console_config_trustROS_retry.png)    
3. 버튼을 누른 후 로봇이 구동되는 소리가 들리거나, 시스템 상에서 활성화 상태가 유지되는지 확인합니다.

:::warning[경고] 
ROBOT MODEL 필드에 <strong>`Robot model을 불러오지 못했습니다.`</strong>라는 문구가 표시된다면 콘솔이 로봇을 찾지 못한 상태입니다. 

1. 로봇 본체의 전원이 물리적으로 켜져 있는지 확인하세요.
2. 입력한 **ROBOT IP**가 정확한지 다시 한 번 확인합니다.
3. 하단의 **[Save Changes]** 버튼을 다시 한번 눌러 설정을 갱신합니다.
![console_config_trustROS_retry2](./image/2.1_console_config_trustROS_retry2.png)
