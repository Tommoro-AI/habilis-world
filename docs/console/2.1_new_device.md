# 2.1 신규 디바이스 등록
## Step 1. 메뉴 접속

좌측 메뉴 바에서 **[Network]** 또는 **[Add Robot]** 탭을 클릭합니다.
![console_config_main_detail](./image/그림7.png)

## Step 3. 핸드 등록
1. Hand 토글을 눌러 IP 주소 입력 필드를 활성화합니다.
2. 활성화된 입력 필드에 사용할 핸드의 IP 주소를 입력합니다.
![Console Main 페이지](./image/그림20.png)

## Step 4. VR 기기 등록
1. VR 토글을 눌러 IP 주소 입력 필드를 활성화합니다.
2. 활성화된 입력 필드에 사용할 VR 기기의 IP 주소를 입력합니다.
![Console Main 페이지](./image/그림22.png)

## Step 5. 인증서 허용    
통신 보안을 위해 최초 1회 인증서 승인이 필요합니다.

1. IP 주소 필드 우측의 **[Trust ROS Cert]** 버튼을 클릭합니다. 
![Console Main 페이지](./image/그림24.png)

2. 브라우저 새 창에 보안 경고가 뜨면 '**고급 > 안전하지 않음으로 이동(Proceed)**'을 선택합니다.
![Console Main 페이지](./image/그림11.png)
![Console Main 페이지](./image/그림12.png)

3. 인증서 페이지가 정상적으로 열리면 창을 닫고 콘솔로 돌아옵니다.  
![Console Main 페이지](./image/그림13.png)

## Step 6. 디바이스 전원 구동

1. <strong> ROBOT MODEL</strong> 필드에 로봇 모델 이름(예: `rby1a`, `ffw_bg2_rev4`)이 정상적으로 표시되는지 확인합니다.
![console_config_trustROS_retry](./image/그림18.png)    
2. 우측의 **[ROBOT ON]** 버튼을 클릭하여 시스템 전원을 켭니다.
![console_config_trustROS_retry](./image/그림19.png)   
3. 핸드를 사용하는 경우, 우측의 **[HAND ON]** 버튼을 클릭하여 핸드 디바이스의 전원을 켭니다.
![Console Main 페이지](./image/그림21.png)  
4. 버튼을 누른 후 로봇이 구동되는 소리가 들리거나, 시스템 상에서 활성화 상태가 유지되는지 확인합니다.

:::warning[경고] 
ROBOT MODEL 필드에 <strong> `Robot model을 불러오지 못했습니다.`</strong>라는 문구가 표시된다면 콘솔이 로봇을 찾지 못한 상태입니다. 
![console_config_trustROS_retry2](./image/그림23.png)

1. 로봇 본체의 전원이 물리적으로 켜져 있는지 확인하세요.
2. 입력한 **ROBOT IP**가 정확한지 다시 한 번 확인합니다.
3. 하단의 **[Save Changes]** 버튼을 다시 한번 눌러 설정을 갱신합니다.
