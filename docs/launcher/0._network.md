# 로봇실 사전 요구사항


## 1. 네트워크
로봇과 PC사이에 ROS통신이 가능하도록 하나의 라우터 아래에 통신에 사용될 모든 노드들을 연결해주세요. 
<br />

[예시. 라우터 1대 - 로봇 1대 - PC 1대]
![Network Setting](./image/0.0_warning_network.png)
<br />

단, 로봇의 ROS Domain ID가 겹치는 일이 없도록 하나의 라우터 아래에는 로봇을 1대만 연결하도록 합니다.
![Network Setting2](./image/0.1_warning_ros_id.png)
<br />

> ROS 통신에서는 IP가 아닌 브로드케스팅을 사용하기 때문에, 동일 네트워크에 여러대 로봇이 연결 될 경우 서로 다른 로봇을 구분하지 못하는 문제가 발생합니다. <strong>통신의 혼선을 피하기 위해 `ROS_ID`(ROS_DOMAIN_ID)를 다르게 설정하여 구분하시기 바랍니다.</strong>

## 3. 런처 위치 권장사항

#### 로봇 패키지 설치
런처에서 로봇이름(예시: `rby1a`)으로 되어있는 패키지는 각 로봇을 위한 SDK+ROS 패키지 입니다. <strong>이 패키지를 로봇 UPC에 설치해야 콘솔에서 로봇을 인식할 수 있습니다.</strong>
<br />

#### 확장팩 설치
콘솔은 로봇을 원격에서 조작할 수 있는 서비스입니다.
만약, AI를 학습하고 추론하거나 Data를 수집하기 위해서는 AI manager나 Data manager를 설치해야합니다. 이러한 추가 기능을 제공하는 모듈을 `확장팩` 이라고 합니다.
<br />

<strong> <설치 예시 1 - 성능이 좋은 로봇 PC를 사용하는 경우> </strong>

설치위치 : 로봇  UPC <br />
설치파일 : `rby1a`, `dataset_manager`, `ai_manager`, `vr_manager`
<br />

<strong> <설치 예시 2 - 성능이 좋지 않은 로봇 PC를 사용하는 경우> </strong>

설치위치 : 로봇  UPC <br />
설치파일 : `rby1a`(필수)
<br />

설치위치 : 확장 PC <br />
설치파일 : `dataset_manager`, `ai_manager`, `vr_manager`
