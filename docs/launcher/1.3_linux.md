# 1.3 리눅스 설치 방법

리눅스 환경에서는 AppImage 실행 파일을 실행하여 하빌리스 런처를 시작합니다.  
하빌리스 런처는 내부적으로 컨테이너 환경을 사용하므로 <strong> Podman 설치가 필요합니다.</strong>

## Step 1. Podman 설치
1. 사용 중인 리눅스 배포판에 맞게 아래 명령어를 실행해 주세요.

#### Ubuntu/Debian 계열
```bash
sudo apt update
sudo apt install -y podman podman-compose
```
#### Ubuntu 22.04
```bash
sudo sh -c "echo 'deb https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/ /' > /etc/apt/sources.list.d/libcontainers.list"
sudo curl -fsSL https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/Release.key | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/libcontainers.gpg
sudo apt update
sudo apt install -y podman podman-compose
```

#### Fedora/CentOS/Rocky/AlmaLinux
```bash
sudo dnf install -y podman podman-compose
```

#### Robot(Jetson & Linux22.04)
```bash
set -e

sudo apt update
sudo apt install -y curl ca-certificates gnupg python3-pip

sudo sh -c "echo 'deb https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/ /' > /etc/apt/sources.list.d/libcontainers.list"

sudo curl -fsSL https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/Release.key \
  | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/libcontainers.gpg

sudo apt update

sudo apt -y \
  -o Dpkg::Options::="--force-confdef" \
  -o Dpkg::Options::="--force-confold" \
  install podman

sudo pip install podman-compose
```

2. 실행 후 정상적으로 설치되었는지 확인합니다.

```bash
podman --version
podman-compose --version
```
podman > 4.x이상 버젼이 설치되어 있는지 확인해주세요

## Step 2. 설치

1. 터미널에서 **다운로드 폴더**로 이동합니다.
```bash
cd ~/Downloads
```

2. 파일 실행 권한을 부여합니다.
```bash
chmod +x habilis-launcher.AppImage
```
:::warning[경고]
- `chmod +x` 명령은 **AppImage 파일이 있는 폴더**에서 실행해야 합니다.
- 파일 탐색기에서 해당 파일을 마우스 우클릭 > [속성] > [권한] 탭에서 <strong> '프로그램으로 실행 허용'</strong>을 체크해도 동일하게 작동합니다.
:::

## Step 3. 하빌리스 런처 실행
1. AppImage를 실행합니다.
```bash
./habilis-launcher.AppImage
```

:::warning[경고]
- Podman이 설치되어 있지 않으면 Launcher가 정상적으로 동작하지 않습니다.
- AppImage 실행 시 오류 메시지가 나온다면 `chmod +x habilis-launcher.AppImage` 를 다시 시도해주세요.
:::
